# Release 0.8.0 — Intelligence (DSS): insights que aconselham, nunca comandam (PromoFxAdvisor + OverrideNudge gated)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 7 (Intelligence / DSS prescritivo)
- **Specs entregues:** SPEC-0013 (novo módulo `intelligence`)
- **Tag git:** `0.8.0`

## Destaques

A Fase 7 torna o ERP **não-burro**: o **12º módulo** `com.fksoft.domain.intelligence` é um
**read-model que escuta eventos de todos os contextos (somente leitura) e ACONSELHA, NUNCA COMANDA**
(redesenho Parte 8). Entrega o **framework de Insight** (evidência + proveniência / recomendação +
ganho-risco / guardrail / decisão humana) e dois insights de lucro direto: o **`PromoFxAdvisor`**
(a promoção de câmbio **se paga**? por agência — veredito **CONVERTE × QUEIMA_MARGEM** com ganho
estimado e **citando as fontes**) e o seam do **`OverrideNudge`** (desligado por **feature flag** até
existir o modelo de faixas — Q4 —, **sem dado falso**). A "inteligência" da v1 é **determinística,
baseada em regra** (Regra Zero): testável, reproduzível, **sem dependência externa**; a porta
`InsightNarrator` deixa o **seam de LLM** pronto (default determinístico, **não wired** — nenhuma
API key no build). O princípio **"aconselha, nunca comanda"** é **provado por ArchUnit com teste de
dentes** e por um e2e que gera o insight a partir de eventos **sem mutar a origem**. **219 testes
verdes**, **12 módulos Modulith acíclicos** (intelligence é folha — ninguém depende dele), 0 violações
de Checkstyle; sem UI nesta fase (backend-first, como Fases 2–6).

## Adicionado / Alterado

- **Added — Framework de Insight + PromoFxAdvisor (fatia 12a):** novo módulo
  `com.fksoft.domain.intelligence` (12º Modulith). Read-model `Insight` (BR1: evidência com
  proveniência, recomendação com `estimatedGain/Risk`, guardrail, status humano). **Consumidor-folha**
  (DL-0034): aprende `booking→account` de `BookingConfirmed` e correlaciona
  `RateSubsidyAccrued`/`FxPositionClosed` por evento, **nunca** chamando de volta um produtor.
  `PromoFxAdvisor` **determinístico** (DL-0035): **CONVERTE** se `volume ≥ 5 ∧ gap ≥ 0`,
  **QUEIMA_MARGEM** se `gap < 0 ∧ |gap| > R$ 1.000`; ganho estimado = gap; saída **validada antes de
  persistir** (BR7, fallback = nenhum insight). Eventos `InsightGenerated`/`InsightDecided`.
  `GET /api/intelligence/insights[/{id}]` (sort por `estimatedGain desc`; 404
  `intelligence.insight.not-found`). **Migração V17**.
- **Added — OverrideNudge (gated) + decisão humana (fatia 12b):** `OverrideEventsListener` consome o
  `PriceOverridden` **exposto** do quoting; `IntelligenceService.onPriceOverridden` **curto-circuita
  sob a flag** `intelligence.override-nudge.enabled` (default `false`) — **sem dado falso** até a
  tabela de faixas existir (BR6/Q4, DL-0036). `POST /api/intelligence/insights/{id}/decision` registra
  a decisão humana (BR4) **sem disparar ação** (BR2); decisão fora de {ACCEPTED, REJECTED, DISMISSED}
  → **400 `intelligence.decision.invalid`**. Observabilidade de IA: `InsightGenerated`/`InsightDecided`
  logados como evento de negócio (insightId, type, subjectRef, correlation id); `decidedBy/decidedAt`
  = métrica **aceitos × rejeitados**.
- **Added — Seam de IA (porta `InsightNarrator`):** default `RuleBasedInsightNarrator` determinístico.
  Um provedor real (futuro) fica **atrás da porta** (ACL) com **stub nos testes**, saída validada,
  provider/model/prompt versionados e dado pessoal mascarado (messaging-and-integrations.md) — id de
  modelo `claude-opus-4-8` para qualquer wiring real. **Não wired** nesta fase (Regra Zero).
- **Changed:** `HttpErrorMapping` (+`intelligence.insight.not-found`, `intelligence.decision.invalid`),
  i18n pt-BR + fallback en, `ArchitectureTest` (+regra `INTELLIGENCE_ADVISES_NEVER_COMMANDS` e teste de
  dentes), `pom.xml` para `0.8.0`.

## Migrações

- `V17__create_intelligence.sql`: `insights` (read-model com evidência/recomendação/guardrail em
  colunas estruturadas — sem jsonb, Regra Zero; UNIQUE `(type, subject_kind, subject_ref)` para upsert
  on-event), `intelligence_booking_attribution` (correlação `booking→account` aprendida de evento +
  buffer incremental, sem double-count) e `intelligence_agency_fx_accrual` (totais por agência).
  **Sem FK cross-contexto** — `subject_ref`/`account_id`/`booking_id` são valores.

## Contratos / API

- `GET /api/intelligence/insights?type=&subjectRef=&status=&page=&size=` → `PageResponse` de
  `InsightView` (sort default `estimatedGain desc`).
- `GET /api/intelligence/insights/{id}` → 200 | 404 `intelligence.insight.not-found`.
- `POST /api/intelligence/insights/{id}/decision` `{decision, note}` → 200 (registra a decisão; **não**
  executa ação) | 400 `intelligence.decision.invalid`.
- **Não há** endpoint que faça o DSS agir sobre a operação — por princípio (BR2).
- Eventos: `InsightGenerated`, `InsightDecided`. Tudo **aditivo** (novo módulo/endpoints/eventos) →
  **MINOR** (ADR 0015). OpenAPI via springdoc; erro estável `{code, message, fields}`.

## Decisões (decision-log)

- **DL-0034** — **PromoFxAdvisor: sujeito = agência** (único eixo derivável de evento,
  `BookingConfirmed.accountId`); `subjectKind` enum plugável p/ rota/produto; intelligence é
  **consumidor-folha** (correlaciona só por evento) — Média/Moderada.
- **DL-0035** — **Advisor determinístico**; limites CONVERTE × QUEIMA_MARGEM (`MIN_VOLUME=5`,
  `BURN_THRESHOLD=R$ 1.000`) + ganho estimado; saída validada + fallback — Média/Barata (limites são
  constantes governadas, graduam p/ SPEC-0014).
- **DL-0036** — **OverrideNudge desligado por feature flag** (Q4); **nenhum LLM wired** (porta
  `InsightNarrator` default determinístico); recomputação **on-event** — Média/Barata.

> **decision-log/INDEX.md:** nenhuma DL desta fase é Confiança=Baixa ou Reversibilidade=Cara — as três
> são **Confiança=Média** e reversão **Moderada/Barata**. A incógnita de negócio (Q4 — faixas de
> override) **permanece em aberto** e fica atrás de flag, sem inventar regra (BR6).

## Riscos e pendências

- **Q4 — modelo de faixas de override (DL-0036):** **em aberto** (incógnita do diretor). O
  `OverrideNudge` fica desligado por flag até a tabela existir; o seam (tipo, listener, branch) está
  pronto para ligar sem refator. **Confirmar com o dono.**
- **Eixo rota/produto do PromoFxAdvisor (DL-0034):** v1 usa **agência** (único derivável de evento). Ligar
  rota/produto exige **enriquecer os eventos produtores** com o eixo; o enum `subjectKind` já comporta.
- **`SpreadRealized` como receita atraída:** não usado (chaveado por `caseId`, sem `bookingId` no
  payload). Enriquecer o evento com `bookingId` é melhoria futura.
- **LLM real:** não wired (DL-0036) — a porta `InsightNarrator` tem default determinístico; um provedor
  entra como adapter de `infra/integration` com stub nos testes, saída validada/versionada e dado
  pessoal mascarado. **O build nunca depende de uma chamada externa.**
- **Métricas Prometheus** da spec (insights gerados/aceitos×rejeitados, distribuição de confiança,
  fallback): por ora **business-event logging** (padrão das Fases 1–6). Ligar o Micrometer é melhoria
  futura.
- **Demais relatórios do catálogo 8.2** (churn, forecast, mix, supplier leverage…): cada um é uma spec
  própria quando priorizado — fora de escopo.
- **Telas Angular:** backend-first nesta fase.
