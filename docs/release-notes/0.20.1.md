# Release 0.20.1 — Limpeza estrutural: remover `internal` do domain (refactor, sem novidade ao usuário)

- **Data:** 2026-06-30
- **Fase do ROADMAP:** 9 (Limpeza estrutural — **ADR + chore**, sem SPEC) — herança Go (pacotes
  planos; encapsulação por tooling, não por pasta)
- **Tipo:** **refactor estrutural**, **PATCH** (ADR 0015) — **sem mudança de contrato**
  (REST/DTO/JSON/evento publicado/i18n/schema) e **sem nova capacidade ao usuário** (Regra Zero)
- **Tag git:** `0.20.1`

## O que mudou (e o que NÃO mudou)

Esta release **não adiciona nenhuma funcionalidade visível ao usuário**. É uma reorganização interna
do código do backend: os tipos de implementação de cada módulo de domínio saíram do subpacote
`com.fksoft.domain.<módulo>.internal.*` e passaram para a base do módulo
`com.fksoft.domain.<módulo>.*`. A convenção de pasta `internal` — herdada da versão Go do produto, onde
`internal/` é uma palavra-chave do compilador — **não existe mais** sob `com.fksoft.domain`.

**Por isso o MANUAL do usuário NÃO muda:** nada na navegação, telas, endpoints, dados ou
comportamento foi alterado. É refactor puro.

## Como a encapsulação continua provada (movemos o mecanismo, não removemos)

Ao achatar, a base de cada módulo passaria a ser, para o Spring Modulith, a API pública inteira — ou
seja, o Modulith **deixaria de esconder** os tipos que antes estavam em `internal`. Para **não
afrouxar nenhum portão** (CLAUDE.md Regra 5), a fronteira foi **movida** de "pasta escondida pelo
Modulith" para:

1. **Marcador de tipo `@com.fksoft.domain.ModuleInternal`** em cada tipo público que era `internal`
   (entidades, repositórios, codecs públicos). Tipos package-private continuam escondidos pelo próprio
   compilador, sem anotação.
2. **Regra ArchUnit `MODULE_INTERNAL_TYPES_ARE_NOT_VISIBLE_ACROSS_MODULES`**: nenhuma classe de
   **outro** módulo de domínio pode depender de um tipo `@ModuleInternal`. O próprio módulo (e seus
   testes) e a camada `infra` (que pode operar a persistência do módulo — ADR 0010/0012) são exceções.
3. **Teeth**: um teste planta a violação (uma classe de um módulo fictício depende de um tipo
   `@ModuleInternal` de outro) e prova que a regra **falha** — a fronteira continua com dentes.

Os predicados de fronteira de **Intelligence** (advises, never commands), **Portfolio** (references,
never commands) e **Platform** (orchestrates, never owns) passaram a reconhecer o marcador
`@ModuleInternal` no lugar do sinal `.internal`.

## Escopo do refactor

- **20 módulos de domínio** achatados (todos os que tinham `internal`); `commissioning` já era plano;
  `money`/`error` são kernels não-módulo.
- **~126 tipos main + ~10 tipos de teste** movidos de `internal/` para a base do módulo (main e test).
- **~106 tipos públicos** marcados `@ModuleInternal`; os ~20 package-private (listeners, codecs,
  guards) seguem sem anotação.
- `package-info.java` de cada módulo atualizado: a prosa descreve a API nomeada + a convenção
  `@ModuleInternal`/ArchUnit no lugar de "o subpacote `internal` é module-private".

## Sem migração, sem contrato

- **Nenhuma migração Flyway** (sem mudança de schema). **Nenhum** endpoint/DTO/JSON/evento
  publicado/chave i18n alterado. Os FQCN dos tipos ex-`internal` mudam de pacote — eles não fazem
  parte de nenhum contrato de fio (injeção por construtor, JPA e Flyway não dependem do pacote).
- OpenAPI: apenas a **versão** do documento sobe para **0.20.1** (metadado, não contrato).

## Decisões (ADR + decision-log)

- **ADR 0016** — achatar `internal`; encapsular por `@ModuleInternal` + ArchUnit (refina ADR 0012/0014).
- **DL-0089** — escolha do mecanismo (marcador de tipo + regra ArchUnit) sobre as alternativas
  (`allowedDependencies`/`@NamedInterface` por subpacote; só package-private; confiar só no Modulith) —
  Alta/Moderada.

## Verificação

- `./mvnw verify` **verde**: **468 testes**, 0 falhas (eram 466 na 0.20.0; +1 teste de teeth da
  nova regra + a própria regra contada como `@ArchTest` — refactor **não adiciona testes de negócio**);
  **ArchUnit 16 regras** (era 15 — +`MODULE_INTERNAL_TYPES_ARE_NOT_VISIBLE_ACROSS_MODULES`);
  **Spring Modulith** acíclico, 22 módulos; **Spotless** 0 alterações; **Checkstyle** 0 violações.
- **Nenhum pacote `internal`** sob `com.fksoft.domain` (main e test) — critério de aceite da Fase 9.

- A **Fase 10** (UX & Frontend profissional, SPEC-0026) é a próxima — a primeira que volta a entregar
