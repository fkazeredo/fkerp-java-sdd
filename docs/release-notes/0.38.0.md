# Release 0.38.0 — Fase 19f · Migração de bibliotecas (Spring Boot 4.1 + Jackson 3)

- **Data:** 2026-07-02
- **Fase do ROADMAP:** 19 (fatia 19f de 12 — Refactoring de maturidade)
- **Specs entregues:** — (chore de stack)
- **Decisões:** DL-0128; ADR-0017 atualizado
- **Tag git:** `0.38.0`

## Destaques

Pedido explícito do dono: **Spring Boot 4.1 + Jackson atualizado**. Quita o débito rastreado da
DL-0108 (a ponte de compatibilidade Jackson 2).

- **Spring Boot 4.0.7 → 4.1.0**; **Spring Modulith 2.0.7 → 2.1.0**; springdoc 3.0.3 mantido; Java 21
  LTS mantido.
- **Jackson 2 → Jackson 3** (`tools.jackson`): removida a ponte `spring-boot-starter-classic`;
  produção agora roda no serializador default do Boot 4. Os 9 arquivos que usavam
  `com.fasterxml.jackson.databind` migraram (`ObjectMapper`/`TypeReference` → `tools.jackson`;
  `JsonProcessingException` checada → `JacksonException` não-checada). As **anotações**
  (`@JsonIgnoreProperties`) ficam em `com.fasterxml.jackson.annotation` — pacote inalterado no
  Jackson 3, DTOs anotados seguem válidos.
- **Flyway modular** (Boot 4 modularizou o auto-config): com a saída do `starter-classic`, o Flyway
  cru deixou de ser auto-configurado (schema vazio → 303 testes falhando); corrigido usando o
  starter modular **`spring-boot-starter-flyway`** (mantendo `flyway-database-postgresql`).
- **Contrato byte-idêntico**: o **snapshot OpenAPI** (drift gate, DL-0126), o **invariante de
  contrato da Fase 18** e a suíte completa provam que o JSON não mudou.

## Alterado

- `pom.xml`: parent 4.1.0, `spring-modulith.version` 2.1.0, remoção do `starter-classic`,
  `flyway-core` → `spring-boot-starter-flyway`.
- 9 arquivos main + 2 de teste migrados a `tools.jackson`; 1 `catch (IOException)` →
  `catch (JacksonException)` (`PaymentWebhookReceiver`).
- ADR-0017 atualizado; DL-0108 quitada.

## Migrações

- Nenhuma (schema inalterado).

## Impacto em OpenAPI / contratos

- Nenhum shape muda; o snapshot `docs/api/openapi.json` permanece igual (só a versão do documento).

## Portões

- Backend `./mvnw verify` verde na stack nova (Boot 4.1.0 / Jackson 3); demais gates verdes.

## Riscos / débito

- Ao sair do `starter-classic`, cada auto-config antes agregado precisa do seu **starter modular**
  próprio (lição do Flyway) — os demais starters explícitos do projeto já cobrem os seus.
