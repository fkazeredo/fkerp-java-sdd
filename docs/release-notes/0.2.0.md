# Release 0.2.0 — Núcleo comercial manual (backend)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 1 (Núcleo comercial manual)
- **Specs entregues:** SPEC-0002, SPEC-0003, SPEC-0004, SPEC-0005, SPEC-0006, SPEC-0007
- **Tag git:** `0.2.0`

## Destaques

O **núcleo comercial 100% manual e rastreável** do ERP, ponta a ponta no backend: cadastrar a conta,
fixar o câmbio congelado, compor a cotação com as duas comissões e o spread, registrar override com
rastro, virar reserva com ciclo de vida, e **conciliar** cada venda — cruzando esperado × realizado
e isolando o ganho/perda cambial. Seis bounded contexts como **monólito modular** (Spring Modulith),
com as fronteiras impostas por testes. **82 testes verdes**; sem UI ainda (ver Pendências).

## Adicionado

- **Accounts (SPEC-0002):** conta comercial CNPJ/MEI/CPF com validação de dígitos (value object
  `Document`), unicidade e status `ACTIVE`. `AccountDirectory` (porta).
- **Exchange (SPEC-0003):** taxa de venda congelada **append-only** (Open-Host), taxa vigente e
  histórico; porta `ExchangeRateProvider`. Taxa futura não é servida antes do tempo.
- **Commissioning (SPEC-0004):** comissão de duas pontas + spread (cálculo puro, HALF_UP), spread
  negativo exposto; porta `CommissionCalculator`. Kernel `Money` compartilhado.
- **Quoting (SPEC-0005, keystone):** composição MANUAL (base BRL + markup, DL-0009), proveniência
  **congelada**, `OverrideRecord` com motivo; consome as 3 fachadas + `MarkupProvider` stub
  (CommercialPolicy). Porta `QuoteDirectory`.
- **Booking (SPEC-0006):** máquina de estados, localizador interno/externo único, **timeout de
  PENDING em 72h** (job), eventos de ciclo de vida.
- **Reconciliation (SPEC-0007):** abre caso ao consumir `BookingConfirmed` (idempotente), registra
  realizados, deriva `realizedSpread`/`fxGainLoss`/discrepância (tolerância DL-0011), lock pessimista.
- **Infra:** handlers globais para corpo malformado e parâmetro inválido (400); `HttpErrorMapping`
  com 18 exceções; `@EnableScheduling`.

## Decisões (decision-log)

- **DL-0007** — Accounts: CADASTUR/IATA opcionais (Média/Barata).
- **DL-0008** — manter nome `Exchange` (Alta/Moderada).
- **DL-0009** — Quoting: preço = base BRL + markup; base comissionável BRL (Média/**Cara**).
- **DL-0010** — Booking: Quote→Booking não 1:1 (Média/Moderada).
- **DL-0011** — Reconciliation: tolerância `max(R$1; 0,5%)` (Média/Barata).
- **ADR 0015** — Versionamento semântico (SemVer): por isso esta entrega é `0.2.0`.
