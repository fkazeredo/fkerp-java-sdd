# Release 0.1.0 — Fundação (Walking Skeleton)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 0 (Fundação)
- **Specs entregues:** SPEC-0001
- **Tag git:** `0.1.0`

## Destaques

Primeira versão: o **monólito modular** do ERP Acme Travel que **compila, sobe, conecta no Postgres,
tem testes verdes e CI**, com as regras de arquitetura travadas por **ArchUnit + Spring Modulith +
Spotless/Checkstyle** desde o primeiro código. Sem regra de negócio (a primeira capacidade — Accounts
— começa na SPEC-0002). Inclui o **Event Storming** da venda Portal de Experiências.

## Adicionado

- **Backend** (`com.fksoft`, 3 camadas — ADR 0012): kernel `domain.error` (`DomainException`
  sem transporte — ADR 0011, `ErrorDetails`, `RateLimited`); `infra.web`
  (`ApiErrorResponse`/`GlobalExceptionHandler`/`HttpErrorMapping`/`PageResponse`);
  `infra.security` (`UserContext` + `UserContextProvider` + **stub de dev** até SPEC-0024);
  `infra.i18n` (pt-BR + fallback); `infra.observability` (correlation id);
  `infra.time` (Clock UTC); `infra.health` (probe de banco); `infra.openapi` (springdoc).
- **API:** `GET /api/system/health` (readiness real, checa o banco) → `{status, db}`; OpenAPI em
  `/v3/api-docs` e Swagger UI.
- **Persistência:** Flyway `V1__baseline.sql` (extensão `pgcrypto`).
- **Frontend** (Angular 22, standalone + signals): `core/http` (base URL, interceptors de
  correlation id e de erro), i18n via ngx-translate (pt-BR/en), tela **health** com estados
  loading/sucesso/erro.
- **Build/Deploy:** Maven Wrapper (Maven 3.9.16), `docker-compose` (app + Postgres), `Dockerfile`
  multi-stage, `.env.example`.
- **CI:** GitHub Actions (backend `verify`, `flyway validate`, frontend lint/test/build).
- **Docs:** `docs/event-storming.md`, `docs/plan/`, `docs/decision-log/` (DL-0001..0006),
  `docs/test-report/`, `docs/ROADMAP-STATUS.md`.

## Como rodar / verificar

```bash
cd backend && ./mvnw verify         # 12 testes verdes + ArchUnit/Modulith/Spotless/Checkstyle
docker compose up --build           # app + db; GET /api/system/health -> 200 {status:UP,db:UP}
cd frontend && npm ci && npx ng lint && npx ng test --watch=false && npx ng build
```
