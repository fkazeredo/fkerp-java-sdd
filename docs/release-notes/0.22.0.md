# Release 0.22.0 — Fase 11 · Observabilidade & monitoramento

- **Data:** 2026-06-30
- **Fase do ROADMAP:** 11
- **Specs entregues:** SPEC-0027 (nova)
- **Tag git:** `0.22.0`

## Destaques

Base de **observabilidade** do ERP, trazida do projeto irmão **fkerp-poc** e adaptada às camadas
(ADR 0012) e ao modelo de papéis (SPEC-0024) deste projeto — **sem mudar nenhuma regra de negócio**
(Regra Zero): instrumenta e expõe o que já existe. **Micrometer + Spring Boot Actuator + registry
Prometheus** (`/actuator/prometheus`), **logs estruturados em JSON** (com correlation id, sem
segredo/PII), **stack de monitoramento** (Prometheus + Loki + Grafana Alloy + Grafana) sob `infra/`
via Docker Compose, **`GET /api/version`** (versão/commit/build) e **métricas de negócio** derivadas
de eventos de domínio **já publicados**.

## Adicionado / Alterado / Corrigido

- **Added:** `micrometer-registry-prometheus`; exposição **deliberada** do Actuator
  (`health,info,prometheus,metrics`) — `env`/`beans`/`heapdump`/etc. **não** expostos (DL-0095).
- **Added:** segurança dos endpoints — `health`/`info`/`/api/version` **públicos**;
  `prometheus`/`metrics` exigem **`ROLE_IT`** (401 anônimo, 403 sem o papel, 200 com TI) (DL-0095).
- **Added:** `GET /api/version` → `{ version, gitCommit, buildTime }` por **build-info** do Spring Boot
  + `git-commit-id-maven-plugin`, com **degradação graciosa** (campos ausentes → `"unknown"`) (DL-0097).
- **Added:** logs **estruturados em JSON** (ECS) ligados no container por
  `LOGGING_STRUCTURED_FORMAT_CONSOLE`; o **correlation id** do MDC vira campo JSON; dev/local segue
  console humano. Higiene reforçada: nenhum segredo/PII logado (DL-0096, BR5).
- **Added:** `infra/` (espelha a POC): `prometheus/`, `loki/`, `alloy/`, `grafana/` (datasources +
  dashboard **"Acme Travel ERP — Backend Overview"** pré-provisionados); serviços de monitoramento no
  `docker-compose.yml` (rede `acme`), `.env.example` e `infra/prometheus/README.md`.
- **Added:** métricas de **negócio** (`BusinessMetrics`, em `infra.observability`) sobre eventos **já
  publicados** — `acme.bookings.confirmed/cancelled`, `acme.quotes.composed/overridden`,
  `acme.billing.invoices.issued`, `acme.finance.periods.closed`, `acme.identity.logins` (DL-0098).
- **Added:** regra **ArchUnit** `DOMAIN_MUST_NOT_DEPEND_ON_METRICS_OR_ACTUATOR` (o domínio não importa
  Micrometer/Actuator — observabilidade é infra).
- **Changed:** `GlobalExceptionHandler` mapeia `NoResourceFoundException` → **404** (URL desconhecida /
  endpoint Actuator não exposto), em vez de 500 (semântica REST correta; i18n `resource.not-found`).
- **Changed:** versão do projeto 0.21.0 → **0.22.0** (pom.xml + OpenAPI info) e a descrição OpenAPI
  ganhou a frase da Fase 11 + o `/api/version`.

## Migrações

- **Nenhuma.** A fase **não muda schema** (próxima migração seria a V31).

## Contratos / API

- **Novo endpoint aditivo:** `GET /api/version` (público; `{version, gitCommit, buildTime}`).
- **Actuator:** `health`/`info` (públicos), `prometheus`/`metrics` (ROLE_IT). Nenhuma mudança de
  contrato de negócio. OpenAPI: `version` → `0.22.0` + frase da Fase 11.

## Decisões (decision-log)

- **DL-0095** — Exposição/segurança do Actuator (health/info/version públicos; prometheus/metrics →
  ROLE_IT; env/beans não expostos). **Média / Barata.**
- **DL-0096** — Logs JSON pelo logging estruturado nativo do Spring Boot (ECS) + higiene de
  mascaramento. **Alta / Barata.**
- **DL-0097** — `/api/version` por build-info + git, com degradação graciosa. **Alta / Barata.**
- **DL-0098** — Métricas de negócio por listener em infra (domínio não conhece Micrometer). **Alta /
  Barata.**

## Como rodar / verificar

```bash
cd backend && ./mvnw verify                 # 477 testes (468 anteriores + 9 novos), gates verdes
cp .env.example .env && touch infra/prometheus/scrape-token
docker compose up --build                   # app+db+prometheus+loki+alloy+grafana
# Grafana http://localhost:3000 (admin/admin) → "Acme Travel ERP — Backend Overview"
curl -s localhost:8080/api/version          # {version, gitCommit, buildTime}
```

> Para o Prometheus raspar `/actuator/prometheus` (protegido por ROLE_IT), gere o token de um usuário
> TI e aponte-o em `infra/prometheus/scrape-token` (ver `infra/prometheus/README.md`).

## Riscos e pendências

- **Token de raspagem expira** (TTL do JWT, 8h): em produção, use um service account/token de longa
  duração e/ou raspagem por rede restrita (a proteção por papel e a por rede se somam — DL-0095).
- **Limiares de alerta** (Alertmanager) e **tracing distribuído** ficam **fora de escopo** (Regra
  Zero): a stack sobe pronta para receber regras; nenhuma é inventada. Tracing só se houver saltos de
  rede a correlacionar (extração de microsserviço) — o correlation id já amarra os logs.
- A verificação ponta a ponta de **métricas/logs no Grafana** é **operacional** (não entra no
  `./mvnw verify` — BR8); foi validada manualmente nesta fase (containers up, datasources e dashboard
  provisionados).
