# Release 0.22.1 — Qualidade & E2E (cobertura como portão + Playwright isolado, sem novidade ao usuário)

- **Data:** 2026-06-30
- **Fase do ROADMAP:** 12 (Qualidade & E2E — **SPEC-0028**)
- **Tipo:** **tooling de teste/CI/cobertura**, **PATCH** (ADR 0015) — **sem mudança de contrato**
  (REST/DTO/JSON/evento publicado/i18n/schema) e **sem nova capacidade ao usuário** (Regra Zero)
- **Tag git:** `0.22.1`

## O que mudou (e o que NÃO mudou)

Esta release **não adiciona nenhuma funcionalidade visível ao usuário**. É a fundação de **qualidade
verificável** do projeto, trazida do projeto irmão **fkerp-poc** e elevada a **portões reais**:

- **Cobertura do backend como portão (JaCoCo):** `./mvnw verify` gera o relatório
  (`target/site/jacoco`) e **falha** se a cobertura de instruções cair abaixo de **80%** (DL-0099). O
  código atual mede **89%** com os 477 testes verdes.
- **Cobertura do frontend como portão (Vitest/v8):** `ng test` coleta cobertura e **falha** abaixo dos
  limiares (statements/lines 65, functions 48, branches 55 — DL-0100). Medido atual: statements 70%,
  lines 72%, functions 54%, branches 60% (app inteiro, 57 testes verdes).
- **Playwright E2E em stack isolado/descartável:** `compose.e2e.yaml` sobe app + **Postgres efêmero
  (tmpfs)** + frontend Nginx em **portas próprias** (4201/8081), distintas do stack de dev — a suíte de
  E2E **NUNCA** toca o banco de desenvolvimento (DL-0101). **11 specs verdes** headless, cobrindo o
  caminho feliz (login → dashboard → navegação → criar conta) e **caminhos tristes** (credencial
  inválida, sem sessão→login, estado vazio, proteção de não-salvos, **401**, **403** por papel).
- **Job de E2E no CI:** `.github/workflows/e2e.yml` sobe o stack isolado, espera a saúde, roda o
  Playwright headless e **sempre** derruba tudo (`if: always()`) — sem tocar dados de dev (DL-0102). O
  `ci.yml` existente segue intacto e agora roda também os gates de cobertura.

**Por isso o MANUAL do usuário NÃO muda:** nada na navegação, telas, endpoints, dados ou comportamento
foi alterado. É tooling interno de qualidade (Regra Zero).

## Prova de isolamento (requisito inegociável)

Durante todo o ciclo `e2e:up` → `e2e` → `e2e:down -v`, o stack de dev (`erpdev`) permaneceu `Exited` e
o volume `erpdev_db-data` **intacto**. O DB do E2E roda em **tmpfs** (sem volume nomeado) e sua porta
**não é exposta ao host** — isolamento por construção (projeto/rede/porta/DB distintos), não condicional.

## Sem migração, sem contrato

- **Nenhuma migração Flyway** (o DB efêmero do E2E roda **as mesmas** migrações do dev). **Nenhum**
  endpoint/DTO/JSON/evento publicado/chave i18n alterado.
- OpenAPI: apenas a **versão** do documento sobe para **0.22.1** (metadado, não contrato).

## Decisões (decision-log)

- **DL-0099** — limiar JaCoCo backend (INSTRUCTION ≥ 80%, gate no `verify`) — Alta/Barata.
- **DL-0100** — limiar Vitest frontend (v8, thresholds no `angular.json`) — Alta/Barata.
- **DL-0101** — isolamento do stack E2E (Postgres efêmero/tmpfs, 4201/8081, perfil dev p/ seed) —
  Alta/Barata.
- **DL-0102** — Playwright (config/jornadas/caminhos tristes) + job de E2E no CI — Alta/Barata.
