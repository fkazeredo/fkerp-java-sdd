# Release 0.3.0 — Compliance mínimo + seam de Finance (backend)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 2 (Compliance mínimo)
- **Specs entregues:** SPEC-0008 (Compliance), SPEC-0015 (Finance — seam mínimo)
- **Tag git:** `0.3.0`

## Destaques

O **documento comprobatório vira cidadão de primeira classe**: um cofre que guarda os documentos
hábeis com hash e prazo legal de guarda, e a regra de ouro do compliance — **lançamento financeiro
sem o documento obrigatório não fecha o mês**. Entrega o **seam mínimo do Finance** (razão AP/AR +
máquina de período) e o **Compliance** (cofre + anexo + retenção + veto de fechamento), com o veto
provado ponta a ponta. Dois bounded contexts novos (Finance, Compliance) no monólito modular. **108
testes verdes**; sem UI ainda (ver Pendências).

## Adicionado / Alterado / Corrigido

- **Added — Finance (SPEC-0015):** `LedgerEntry` (razão AP/AR, máquina PROVISIONAL→CONFIRMED→SETTLED),
  `AccountingPeriod` (OPEN→CLOSING→CLOSED) e `closePeriod` que consulta o Compliance e **falha** se
  houver pendência. Razão em **moeda original** (DL-0013), total do período por moeda. Porta
  `LedgerDirectory` (lida pelo Compliance) e porta `CloseGuard` (consumida; default permissivo
  rastreável `AlwaysAllowsCloseGuard`). Eventos `LedgerEntryRegistered`/`PeriodClosed`.
- **Added — Compliance (SPEC-0008):** cofre `Document` (hash **SHA-256** + `retentionUntil` calculado
  na ingestão pela `RetentionPolicy`), `DocumentAttachment` (vínculo por valor, idempotente),
  `DocumentRequirement` (seed da tabela 7.7, DL-0012), porta `FileStorage` + adaptador
  `FilesystemFileStorage` (DL-0015), `close-check` (o veto, exposto e usado como `CloseGuard
  @Primary`), bloqueio de expurgo antes da retenção, e job `RetentionExpiryScheduler`
  (`RetentionExpiring`). Eventos `DocumentAttached`/`RequirementUnmet`/`RetentionExpiring`.
- **Changed:** `HttpErrorMapping` ganhou 9 exceções novas (Finance + Compliance); `OpenApiConfig`
  para `0.3.0`. DI cycle Finance↔Compliance quebrado com `@Lazy` no `ComplianceCloseGuard`.

## Migrações

- `V7__create_finance.sql`: `accounting_periods`, `ledger_entries` (+ índice por período/status).
- `V8__create_compliance.sql`: `documents`, `document_attachments` (UNIQUE doc+entry),
  `document_requirements` + **seed** dos requisitos (DL-0012) e índice de retenção.

## Contratos / API

- `/api/finance/entries` (+ `/{id}/confirm`), `/api/finance/periods/{yyyymm}` (+ `/close`),
  `/api/compliance/documents` (multipart, + `/{id}/attach`, `/{id}/content`),
  `/api/compliance/close-check`, `DELETE /api/compliance/documents/{id}`. OpenAPI via springdoc.
  Erro estável `{code,message,fields}` (a 409 `finance.period.cannot-close` lista os pendentes em
  `fields`); i18n pt-BR + fallback. **Retrocompatível** (só adições) → MINOR (ADR 0015).

## Decisões (decision-log)

- **DL-0012** — Catálogo `entryType × DocumentRequirement` (seed da tabela 7.7, com fase) — Média/Barata.
- **DL-0013** — Finance: razão em moeda original; total por moeda — Média/**Moderada**.
- **DL-0014** — Finance: construir o seam mínimo; contabilidade plena = comprar depois — Alta/Moderada.
- **DL-0015** — Compliance: porta `FileStorage` + adaptador filesystem; hash SHA-256 — Alta/Barata.

## Como rodar / verificar

```bash
cd backend && ./mvnw verify        # 108 testes + ArchUnit/Modulith(9)/Spotless/Checkstyle (Docker up)

```

## Riscos e pendências

- **Telas Angular** de Finance/Compliance — **pendentes** (backend completo e testado).
- `AT_SETTLEMENT` (comprovante na liquidação) está no seed mas só será cobrado quando o **Payout
  (SPEC-0017)** chegar; o veto de fechamento cobra apenas `AT_REGISTRATION`.
- Cofre usa adaptador **filesystem** (single-instance, ADR 0002); a porta permite S3/Azure depois.
- Eventos in-process; viram outbox/contrato quando cruzarem serviço.
- Open Questions adiadas (na spec): versão/substituição de documento; carimbo do tempo próprio.
