# Release 0.36.0 — Fase 19d · Documentação de API real (springdoc)

- **Data:** 2026-07-02
- **Fase do ROADMAP:** 19 (fatia 19d de 12 — Refactoring de maturidade; pedido explícito do dono)
- **Specs entregues:** SPEC-0024 (segurança/OIDC na doc); `architecture/modules-and-apis.md`
- **Decisões:** DL-0126
- **Tag git:** `0.36.0`

## Destaques

O springdoc já estava na stack mas **vazio**: nenhuma anotação, Authorize inútil, e um changelog de
~200 linhas dentro do código. Esta fatia torna a doc de API **real e usável**.

- **`@Tag` em todos os 37 controllers** (nome de negócio + descrição) → o Swagger UI agrupa a API
  por contexto. **Fitness function** (ArchUnit): todo `@RestController` precisa de `@Tag` — novo
  controller sem tag quebra o build.
- **Contrato de erro documentado globalmente** (`GlobalErrorResponsesCustomizer`): o envelope
  estável `{ code, message, fields }` (ADR 0011) + 400/401/403/404/409/422 em toda operação, sem
  repetir `@ApiResponse` em ~75 endpoints.
- **Botão Authorize funcional**: OAuth2 **Authorization Code + PKCE** apontando ao AS self-hosted
  (client `acme-erp-web`, ADR-0018) — o operador loga na própria Swagger UI e testa um endpoint
  gated. Um `bearerAuth` complementar permite colar um token.
- **`Info.description` enxuta**: o changelog voltou para `docs/release-notes/` (fonte única).
- **Snapshot de contrato versionado** (`docs/api/openapi.json`, ~478 KB) + **teste de drift**
  (`OpenApiSnapshotIntegrationTest`): mudar o contrato sem atualizar o snapshot **falha o build**;
  regenera com `-Dopenapi.snapshot.write=true`. Generaliza o invariante de contrato da Fase 18 à
  API inteira.

## Adicionado

- `@Tag` (37 controllers); `infra.openapi.GlobalErrorResponsesCustomizer`;
  `OpenApiSnapshotIntegrationTest`; regra ArchUnit `REST_CONTROLLERS_ARE_TAGGED_FOR_OPENAPI` (18ª);
  `docs/api/openapi.json` (contrato versionado).

## Alterado

- `OpenApiConfig` reescrito (PKCE + info enxuta); versão do contrato → 0.36.0.

## Migrações

- Nenhuma.

## Impacto em OpenAPI / contratos

- O **conteúdo** da doc melhora muito; nenhum endpoint/JSON muda de shape (o snapshot é o espelho
  fiel do que já existe). Só a `info.version` do documento acompanha a release.

## Portões

  demais gates verdes.
