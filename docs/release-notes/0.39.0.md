- **Data:** 2026-07-02

- **Specs entregues:** SPEC-0024 (AS self-hosted)
- **Decisões:** **ADR-0020** (multi-instance ready — revisa o ADR-0002) + DL-0129
- **Tag git:** `0.39.0`

## Destaques

O app vira **stateless por instância**: o estado de autenticação sai da memória de processo e vai
para o Postgres/configuração. Efeito imediato mesmo em single-instance: **um restart não desloga
mais ninguém**; efeito estrutural: duas instâncias atrás de um load balancer passam a ser possíveis
(prova operacional com reverse proxy fica na 19l).

- **Chave de assinatura persistida** (`PersistedJwk`): `OIDC_JWK_PRIVATE_KEY` (base64 DER PKCS#8) +
  `OIDC_JWK_KEY_ID` estável → todas as instâncias publicam o MESMO JWKS. Sem config → efêmera
  (dev/test); **produção exige a persistida** (check novo no `ProdReadinessValidator`).
- **Estado do AS em JDBC** (migração **V39**, DDL padrão do SAS ajustado a Postgres):
  `JdbcRegisteredClientRepository` (client SPA bootstrapado **idempotentemente**) +
  `JdbcOAuth2AuthorizationService` — qualquer instância completa um fluxo que outra iniciou.
- **Sessão do form-login via Spring Session JDBC** (tabelas `spring_session*` na V39;
  `initialize-schema=never` — o schema é do Flyway). Sem sticky session.
- **Mantidos por instância (decisão consciente):** circuit breakers de saída e o dispatcher do
  mock de pagamento (corrida entre réplicas é no-op — receiver idempotente). Jobs já eram HA-safe
  (advisory lock, DL-0075).
