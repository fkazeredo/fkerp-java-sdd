# Release 0.4.0 — Primeira integração real (ACL do site de cotação, ramo INTEGRATED)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 3 (Primeira integração real — ACL)
- **Specs entregues:** SPEC-0009 (Sourcing + Integration); ativa o ramo INTEGRATED de SPEC-0005
- **Tag git:** `0.4.0`

## Destaques

A **primeira Anti-Corruption Layer de verdade**: o site de cotação entrega um **preço fechado** que
cria uma cotação no ERP **sem rodar o motor de sugestão** (`trustExternalPrice`) — ativando o gancho
que ficou **adormecido** na SPEC-0005. Formaliza o `Sourcing` (de onde a oferta vem) e prova o caminho
**INTEGRATED** de ponta a ponta: webhook assinado (HMAC-SHA256), tradução do payload externo na borda
(o DTO externo **nunca** cruza para o domínio), idempotência por `externalQuotationId`, e a saúde do
conector observável. Um bounded context novo (**Sourcing**, 10º módulo). **135 testes verdes**; sem UI
nesta fase (webhook é máquina-a-máquina).

## Decisões (decision-log)

- **DL-0016** — Webhook: assinatura HMAC-SHA256 + segredo compartilhado (`X-Signature`) — Média/Moderada.
- **DL-0017** — Inbound: Account inexistente **rejeita 422** (não cria provisória/curadoria) —
  **Baixa**/Moderada (decisão de negócio em aberto na SPEC-0009).
- **DL-0018** — Quote INTEGRATED reusa o agregado; colunas de composição MANUAL viram nulas —
  Alta/Moderada.
- **DL-0019** — ACL de entrada: classificação de falha + observabilidade, **sem** circuit breaker (não
  há chamada de saída) — Alta/Barata.

## Riscos e pendências

- **Recomposição do preço integrado:** adormecida por decisão de produto (redesenho Parte 4.3); o
  caminho de reversão é aditivo (DL-0018).
- **`IntegratedQuoteCreated` sem consumidor:** o `Intelligence` (funil por canal, 8.2-F) ainda não
  existe; evento publicado in-process — vira contrato/outbox quando outro serviço consumir.
- **Resiliência de saída** (timeout/retry/circuit breaker): fora de escopo nesta ACL **de entrada**
  (DL-0019); entra com a primeira ACL **de saída** (GDS / crawler de ponto, SPEC-0012).
- **Política de evolução do contrato** do site de cotação: a confirmar com o parceiro (versão hoje é v1
  implícita no path).
- **Conta inexistente no inbound** (DL-0017): decisão de negócio de **Confiança Baixa**; se o dono
  preferir "criar provisória" ou "curadoria", a troca é localizada na resolução da Account.
- **Telas Angular:** o webhook é máquina-a-máquina; sem jornada de tela nesta fase.
