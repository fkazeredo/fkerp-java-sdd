## Destaques

Os adaptadores de integração eram **mocks in-process** (Java puro) — nunca exercitavam HTTP real,
então timeout/retry/circuit-breaker ficavam sem cobertura de verdade. Esta fatia gradua um
**vertical slice representativo** (o outbound de NFS-e municipal) para HTTP real, com emulador
rodando **em dev e nos testes** (pedido do dono), e documenta o padrão para os demais.

- **Adaptador HTTP real** `HttpMunicipalNfseService`: `RestClient` com connect/read timeout,
  **retry** limitado dos transientes (TIMEOUT/UNAVAILABLE; REJECTED é terminal), guardado por um
  **`OutboundCircuitBreaker`** reutilizável (generaliza o breaker in-process do crawler, DL-0031;
  sem resilience4j — Regra Zero). Classificação de falha (BR7): timeout→TIMEOUT, 5xx→UNAVAILABLE,
  422→REJECTED; nunca um "emitido" falso.
- **Seleção por config** `billing.nfse.adapter`: **default `simulated`** (o mock atual segue o
  default — nada quebra); `http` liga o adaptador real. Ambos `@ConditionalOnProperty`.
- **Emulador em DEV** (não só testes): serviço **WireMock** no `docker-compose` sob profile
  `emulators` (`docker compose --profile emulators up`), com fault injection por `municipalityCode`
  (`REJECT`/`TIMEOUT`/sucesso) — `infra/wiremock/` + README. **Em testes**: um `HttpServer` do JDK
  (sem dependência nova) exercita sucesso/rejeição/timeout/breaker.
- **Seam documentado**: os outros mocks (portais/GDS/gateway/newsletter/ponto) graduam pelo mesmo
  molde; o município real (XML/SOAP, e-CNPJ, homologação) segue Open Question.

## Adicionado

- `infra.integration.nfse.HttpMunicipalNfseService` + `MunicipalNfseHttpMessages` (DTO externo).
- Config `billing.nfse.*` (adapter/base-url/timeouts/retries/breaker) no `application.yml`.
- `docker-compose.yml`: serviço `nfse-emulator` (WireMock) no profile `emulators`; `infra/wiremock/`
  (4 mappings + README).
- Testes: `MunicipalNfseHttpAdapterTest` (5 casos: sucesso/rejeição/timeout+retry/breaker-open/
