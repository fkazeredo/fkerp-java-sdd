# Release 0.21.0 — Fase 10 · UX & Frontend profissional

- **Data:** 2026-06-30
- **Fase do ROADMAP:** 10
- **Specs entregues:** SPEC-0026
- **Tag git:** `0.21.0`

## Destaques

Renovação completa da experiência do frontend, sem mudar nenhuma regra de negócio nem contrato de
API: **shell SaaS** (barra lateral + topo + gaveta no celular), **tema claro/escuro**, **paleta de
comandos `Ctrl/Cmd+K`** com atalhos e ajuda, **login** com **revalidação silenciosa de sessão**,
**proteção de não-salvos** (`canDeactivate`), **estados reais** (carregando/vazio/erro/permissão) em
todas as telas e um **Painel (dashboard) com KPIs**. Stack graduado para **PrimeNG 21 (Aura) +
Tailwind v4** sobre Angular 22 — gradua a **DL-0003** (CSS plano da Fase 0).

## Adicionado / Alterado / Corrigido

- **Added:** PrimeNG 21 (preset Aura via `@primeuix/themes`) + primeicons + `@angular/cdk` +
  `@angular/animations`; Tailwind v4 (`@tailwindcss/postcss`) integrado por **camadas CSS**.
- **Added:** `Shell` SaaS (sidebar/topbar/drawer responsivo) como rota de layout; `ThemeService`
  (`.app-dark` + tokens `--app-*`, persistido, default = preferência do SO).
- **Added:** `CommandRegistry` + `ShortcutService` (paleta `Ctrl/Cmd+K`, navegação `g`+tecla, ajuda
  `?`; atalhos de letra ignorados em campos editáveis); componentes `CommandPalette` e `KeyboardHelp`.
- **Added:** revalidação silenciosa de sessão no boot e perto da expiração via `GET /api/identity/me`
  (sem refresh token — DL-0092); guard/interceptor preservam `returnUrl`.
- **Added:** `canDeactivate` (proteção de não-salvos) em Contas/Câmbio/Cotação.
- **Added:** componente `ScreenState` (loading/empty/error/permissão) e repaginação de **todas** as
  telas (Contas, Câmbio, Cotação, Reserva, Conciliação, Saúde, Login) com PrimeNG.
- **Added:** **Painel** com KPIs (Contas/Reservas/Conciliação/Câmbio) calculados no cliente a partir
  dos endpoints de lista existentes (DL-0094); rota raiz protegida `''` → `dashboard`.
- **Changed:** rotas das features convertidas para lazy `loadComponent` (bundle inicial enxuto);
  `BookingService.list()` adicionado (o endpoint `GET /api/bookings` já existia).
- **Changed:** versão do projeto 0.20.1 → **0.21.0** (pom.xml + OpenAPI info).

## Migrações

- Nenhuma. A fase é **frontend-only**; o schema do banco não mudou (próxima migração seria a V31).

## Contratos / API

- OpenAPI: **nenhum endpoint novo** e **nenhuma mudança de contrato**. Só a `version` subiu para
  `0.21.0` e a descrição ganhou uma frase sobre a camada de UX da Fase 10.


## Riscos e pendências

- **PrimeNG 21 declara peer-deps no Angular 21** (a major do PrimeNG fica uma atrás da do Angular):
  resolvido com `legacy-peer-deps=true` no `.npmrc` (runtime compatível). Revisar na Fase 14 quando
  sair o PrimeNG 22.
- **"Silent refresh" é revalidação**, não renovação de token (o backend do 8k não tem refresh token);
  o refresh real chega com o OIDC externo na **Fase 13** (DL-0092, Confiança Média).
- E2E das jornadas (Playwright) fica para a **Fase 12**; observabilidade do front para a **Fase 11**.
