# Release 0.29.0 — Fase 18a · Módulo `cadastro` + enums de referência → cadastros (Admin/Assets/Billing)

- **Data:** 2026-07-01
- **Fase do ROADMAP:** 18 (fatia 18a de 4)
- **Specs entregues:** **SPEC-0031** (nova — registro de dados de referência / cadastro)
- **Decisões:** **ADR-0019** (padrão enum→cadastro) + **DL-0115** (valor persistido vira `code` validado por porta)
- **Tag git:** `0.29.0`

## Destaques

O dono decidiu que **todo enum de referência que não seja máquina de estado nem imutável por lei vira
cadastro** — dado editável (rótulo, ordem, ativação) sem redeploy. Esta fatia (18a) entrega o
**mecanismo** e converte o **primeiro grupo** (Admin/Assets/Billing):

- **Novo módulo `cadastro` (23º Modulith), folha:** uma tabela única `cadastro_item(type, code,
  label, active, sort_order, …)` com `unique(type, code)`, o `CadastroService` (CRUD — `code`
  imutável; `label`/`active`/`sort_order` editáveis) e a **porta pública `CadastroValidator`** que os
  demais módulos usam para validar um `code`.
- **A invariante (contrato de fio inalterado):** o campo `@Enumerated` vira um **`String code`** cujo
  valor **= o nome do antigo constante do enum** (`"UTILITY"`, `"SIMPLES_NACIONAL"`, …). O JSON de
  request/response **não muda**; um `code` novo (sem lógica cablada) **funciona como dado puro**.
- **Lógica que ramificava preservada** por constantes de `code`: `AdminExpenseCodes.entryTypeFor`
  (kind→EntryType→documento, DL-0085), `TaxRegimeCodes` (regime→estratégia de imposto, DL-0044),
  `WithholdingKindCodes` (codec de retenções), `AssetCodes.isSoftwareLicense` (exige expiração).

## Adicionado

- **`GET /api/cadastro/types`** e **`GET /api/cadastro/items?type=…`** (leitura — autenticado).
- **`POST` / `PUT` / `DELETE /api/cadastro/items(/{id})`** (escrita — **`ROLE_POLICY_ADMIN`**,
  reusando um papel existente; nenhum papel novo, o auth não foi tocado após a Fase 17).
- **Tela "Cadastros"** no shell (nav + rota gated por `ROLE_POLICY_ADMIN`): lista os tipos e, do tipo
  escolhido, lista/adiciona/edita/ativa/desativa itens, com `<app-screen-state>` (loading/empty/
  error/permission). i18n pt-BR + en.
- **Migração V33** cria `cadastro_item` e **semeia** os valores atuais dos enums convertidos
  (`code`=nome do enum, `label` pt-BR), idempotente (`ON CONFLICT DO NOTHING`).

## Alterado

- **Admin/Assets/Billing:** os campos de enum (`AdminExpenseKind`, `AdminRecurrence`,
  `AdminSupplierType`, `AssetType`, `WithholdingKind`, `TaxRegime`) passaram a `String code` validado
  contra o cadastro (entidades, DTOs, views, eventos, repositórios, serviços). Os enums foram
  removidos; as constantes que a lógica precisa vivem em classes `*Codes`.
- **OpenAPI** (versão → 0.29.0) documenta os novos `/api/cadastro/*`; **os campos convertidos mantêm
  o schema `string`** — nenhum contrato `/api` existente mudou.

## Removido

- Os enums `AdminExpenseKind`, `AdminRecurrence`, `AdminSupplierType` (módulo `admin`), `AssetType`
  (módulo `assets`), `WithholdingKind`, `TaxRegime` (módulo `billing`) — substituídos por `code`
  validado + constantes `*Codes`.

## Migrações

- **V33__create_cadastro.sql** — cria `cadastro_item` (unique `(type, code)`) e semeia os valores dos
  enums convertidos (Admin/Assets/Billing). Idempotente; nunca edita uma migração aplicada.

## Impacto em OpenAPI / contratos

- **Novos** endpoints `/api/cadastro/*`. Os campos convertidos continuam `string` no JSON — **nenhum
  contrato existente quebrou** (invariante DL-0115). Versão do contrato → **0.29.0**.

## Riscos e o que fica para as próximas fatias (18b–18d)

- **Fallback de código não cablado:** um `code` novo em `ADMIN_EXPENSE_KIND` sem mapa cai em
  `OTHER_EXPENSE` (seam documentado, DL-0115); regimes/retenções novos exigem uma estratégia plugada.
- **Sem trava de obrigatoriedade** (ex.: impedir desativar o último item ativo de um tipo) — Open
  Question da SPEC-0031, adiada até o dono exigir.
- **18b–18d** convertem os demais grupos (Compliance/People/Intelligence/Portfolio/Booking/Payout/
  AfterSales…) reusando este módulo, uma fatia por grupo (`0.30.0`–`0.32.0`).
