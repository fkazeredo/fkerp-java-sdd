# Release 0.2.1 — Telas da Fase 1 (frontend Angular)

- **Data:** 2026-06-29
- **Fase do ROADMAP:** 1 (Núcleo comercial manual) — fecha a fase **ponta a ponta**
- **Specs entregues:** SPEC-0002…0007 (camada de UI)
- **Tag git:** `0.2.1`

## Destaques

Completa a Fase 1 com as **telas Angular** dos seis contextos do núcleo comercial, consumindo as
APIs já entregues em `0.2.0` (sem mudança de contrato — por isso **PATCH**). Cada tela tem os estados
**loading / vazio / erro** exigidos nos acceptance criteria, e o erro é exibido pelo **code estável**
do backend. Navegação entre as telas no topo. CSS puro (PrimeNG/Tailwind seguem adiados, DL-0003).

## Adicionado (5 telas + shell)

- **Accounts:** formulário de criação (tipo legal/documento/nome/CADASTUR/IATA) + lista com filtro por
  status; mostra duplicidade/documento inválido pelo code do backend.
- **Exchange:** "fixar taxa" (par/taxa/nota) + histórico por par.
- **Quoting (keystone):** compor a venda e ver **sugerido × aplicado**, a decomposição de comissão, o
  spread (com selo positivo/negativo), o markup e a **proveniência**; aplicar **override com motivo** e
  ver o histórico de overrides.
- **Booking:** criar a partir do `quoteId` (localizador interno/externo) e detalhe com **apenas as
  ações permitidas pelo estado** (pending/confirm/cancel/no-show/complete/change); cancelar pede motivo.
- **Reconciliation:** lista de casos **priorizada por discrepância** + formulário de liquidação que
  mostra spread realizado, ganho/perda cambial e status.
- **Shell:** navegação (`nav`), rotas (`/accounts` default, `/exchange`, `/quotes`, `/bookings`,
  `/reconciliation`, `/health`), i18n pt-BR + en para todas as telas.

## Testes

`cd frontend && npm` → **lint** ✅ · **test** ✅ (Vitest, **14** casos em 7 arquivos) · **build** ✅
(bundle gerado). Cada tela tem teste de estado de sucesso e de erro (serviço mockado, como a tela de
health da Fase 0).

## Contratos / API

Sem mudança: as telas consomem os endpoints de `0.2.0`. Versão bumpada no `backend/pom.xml`
(fonte da verdade, ADR 0015) para alinhar o produto a `0.2.1`.

## Como rodar / verificar

```bash
cd frontend && npm ci && npx ng lint && npx ng test --watch=false && npx ng build
docker compose up --build      # app + db; navegue por /accounts, /exchange, /quotes, ...
```

## Riscos e pendências

- PrimeNG/Tailwind ainda não adicionados (telas em CSS puro) — entra quando a UI exigir (DL-0003).
- Mensagens de erro exibem o **code** do backend quando não há tradução específica da mensagem
  (mesmo comportamento da tela de health).
