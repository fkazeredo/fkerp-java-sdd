# Caderno de Testes — Fatia 8l-1 (Admin: fornecedores + contratos)

> Spec: **SPEC-0025** (Admin — Fornecedores e Contratos Administrativos). Migração **V30**. Decisões:
> DL-0084 (módulo próprio, registro enxuto), DL-0087 (alerta por relógio controlado — suporte),
> DL-0088 (escritas exigem ROLE_FINANCE + auditoria). Release alvo **0.20.0**.

## Escopo

Cadastro **enxuto** de fornecedor administrativo (UTILITY | SOFTWARE | SERVICE | OTHER) e seus
contratos (vigência, recorrência, valor, documento no Compliance por **valor**), com o gate de papel
e a auditoria de alteração. Cobre os **Acceptance Criteria** relativos ao cadastro/consulta:

- Registrar fornecedor de energia (BR1) — nasce ACTIVE, publica `AdminSupplierRegistered`, auditado.
- Registrar contrato vinculado a documento do Compliance por valor (BR2); janela inválida → 400.
- Escritas exigem `ROLE_FINANCE` (BR11/DL-0088) — 403 + auditoria sem o papel; 201 com.

(A despesa→Finance é a 8l-2; o alerta de contrato a vencer é a 8l-3.)

### Arquitetura (no suíte global)
- **ArchUnit** (`ArchitectureTest`, 15 regras): domínio não depende de application/infra; @Entity sem
  setters/`@Data`; sem `*Impl`; injeção por construtor. O módulo `admin` respeita todas.
- **Spring Modulith** (`ModularityTests.verifiesModularStructure`): grafo acíclico com o **22º módulo
  de negócio** `Admin` (folha; usa `finance`/`compliance`/`platform`/`money`/`error`). Verde.
- **Completeness do erro** (`HttpErrorMappingCompletenessTest`): as 5 exceções de Admin têm mapping
  HTTP (404/400/400/400/409). Verde.
