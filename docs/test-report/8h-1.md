# Caderno de Testes — Fatia 8h-1 (Assets: registro + ciclo de vida)

> Spec: **SPEC-0021** (Assets — Patrimônio Interno). Migração **V26**. Decisões: DL-0064 (contexto
> separado), DL-0065 (sem depreciação/gestão plena), DL-0067 (folha publica eventos), DL-0068 (baixa
> auditada inline; RETIRED terminal). Release alvo **0.16.0**.

## Escopo

Registro de patrimônio interno (equipamento, licença de software, outros bens) com vínculos por
**valor** (documento no Compliance, lançamento no Finance) e ciclo de vida **ACTIVE → RETIRED**
auditado. Cobre os **Acceptance Criteria** da spec relativos ao registro/consulta/baixa:

- Registrar uma licença com vencimento e custo, vinculada a documento e lançamento (BR1/BR2).
- Erro de licença sem `expiresAt` (BR1 / Error Behavior `assets.license.expiry-required`).
- Baixa auditada (BR4) e terminal (DL-0068).
- Assets **não** entra no fluxo comercial (BR5 — não há endpoint de preço/venda).

(O alerta de licença a vencer e a listagem `?expiringWithinDays` são cobertos na fatia 8h-2.)
