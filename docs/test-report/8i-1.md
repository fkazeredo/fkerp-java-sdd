- **Spec:** SPEC-0022 (BR1)
- **Branch:** `feature/8i-1-employee` → `feature/8i-integration`

## Escopo (Acceptance Criteria cobertos)

- "Cadastrar colaborador" — `POST /api/people/employees` cria `Employee` ACTIVE com jornada
  contratada `HH:mm`; identificador único; status `ACTIVE|ON_LEAVE|TERMINATED`.
- `./mvnw verify` verde (ArchUnit + Spring Modulith + Checkstyle/Spotless).

## Casos de teste

### Unitário/domínio — `ContractedJourneyTest` (3 casos)
| Caso | Verifica | Regra |
|---|---|---|
| `parsesAndRendersHhMm` | parse `08:00`→480 e render 480→`08:00` | BR1 (jornada contratada) |
| `rejectsMalformedOrOutOfRangeJourney` | `null`/`8h`/`08:60`/`0`/`>24h` → `EmployeeInvalidException` | BR1 (invariante) |
| `formatsSignedTimeBankBalance` | `+00:20` / `-01:10` / `176:20` | DL-0070 (formato do saldo) |

### Integração (Testcontainers + Postgres) — `EmployeeApiIntegrationTest` (5 casos)
| Caso | Verifica | Regra |
|---|---|---|
| `registersAnActiveEmployeeReadableViaTheApi` | 201 ACTIVE + 200 na leitura; `contractedJourney=08:00` | BR1 |
| `rejectsADuplicateIdentifierWith409` | identificador repetido → 409 `people.employee.duplicate` | BR1 (único) |
| `rejectsAMalformedContractedJourneyWith400` | `"8 horas"` → 400 `people.employee.invalid` | BR1 |
| `unknownEmployeeIdIs404` | id inexistente → 404 `people.employee.not-found` | Error Behavior |
| `listsFilteringByStatus` | `?status=ACTIVE` lista; `?status=TERMINATED` não | API Contracts |

### Arquitetura/portões
- `ArchitectureTest` (14), `ModularityTests` (Modulith acíclico), `HttpErrorMappingCompletenessTest`

## Cobertura — o que NÃO está coberto (e por quê)
- Jornada/banco de horas/divergências → fatia **8i-2**.
- Arquivo de holerite no Compliance → fatia **8i-3**.
- Transições de status do colaborador (ON_LEAVE/TERMINATED) — Out of Scope do v1 (cadastro nasce
  ACTIVE; o ciclo completo de RH é "comprar/integrar").
